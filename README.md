# TicTacToe

Онлайн-версия игры "Крестики-нолики" с ранжированием и рейтингом игроков.

![ICON](https://github.com/AkiyamaGG/TicTacToe/blob/master/icon.png)

## О проекте

Проект состоит из двух основных частей:
- `xo-server` — серверная часть на FastAPI с SQLite, WebSocket матчами и хранением статистики игроков.
- `xo-client` — клиентское приложение на PySide6, которое подключается к серверу и позволяет играть против другого пользователя онлайн.

## Основные возможности

- Онлайн-матчи через WebSocket
- Очередь поиска соперника
- ELO-рейтинг и статистика побед/поражений/ничьих
- Таблица рейтинга по ELO, победам, поражениям и количеству матчей
- Сохранение данных в SQLite
- Управление состоянием игроков (`online` / `offline`)
- Грейсфул остановка сервера и очистка очередей

## Структура проекта

- `xo-server/`
  - `main.py` — запуск сервера и консольные команды
  - `routers/player.py` — API для игроков
  - `routers/rating.py` — API для рейтинга
  - `routers/match.py` — WebSocket-логика матчей и игрового процесса
  - `database/create_database.py` — создание SQLite базы данных
  - `compose.yaml` — Docker Compose для запуска сервера
  - `Dockerfile` — контейнеризация сервера
- `xo-client/`
  - `xo-client.py` — PySide6 клиент для игры
- `README.md` — документация проекта

## Установка

1. Убедитесь, что установлен Python 3.14.3
2. Установите зависимости для сервера и клиента:

```bash
python -m pip install fastapi uvicorn python-dotenv
python -m pip install PySide6 requests websocket-client
```

> Если в проекте есть корректный `requirements.txt`, можно использовать:
> `python -m pip install -r requirements.txt`.

## Запуск сервера

1. Перейдите в папку сервера:

```bash
cd xo-server
```

2. Создайте файл `.env` рядом с `main.py` и добавьте:

```env
PORT=8000 #<-- этот порт используется как пример
```

3. Запустите сервер:

```bash
python main.py
```

4. Сервер будет доступен по адресу:

```
http://127.0.0.1:8000
```

## Запуск клиента

1. Перейдите в папку клиента:

```bash
cd xo-client
```

2. Запустите GUI-приложение:

```bash
python xo-client.py
```

3. Клиент автоматически подключится к серверу по адресу `127.0.0.1:8000`.

## Docker

Для запуска сервера в контейнере используйте Docker Compose:

```bash
docker compose up --build
```

Docker запускает сервис на порту `8000` и монтирует папку `database`.

## API сервера

### Игроки

- `POST /player/create/` — создать нового игрока
  - Тело: `{"nickname": "имя"}`
- `GET /player/info/uid/{user_id}` — получить информацию по UID
- `GET /player/info/nickname/{username}` — получить информацию по нику
- `GET /player/status/uid/{user_id}` — получить статус игрока
- `POST /player/status/uid/update/{user_id}` — обновить статус игрока
  - Тело: `{"status": "online"}` или `{"status": "offline"}`

### Рейтинг

- `GET /rating/{type}/?limit=100`
  - `type` может быть `elo`, `wins`, `loses`, `matches`
  - Возвращает таблицу топ-Игроков по выбранному критерию
  - `?limit` - query параметр - можно менять

### Матчи

- `GET /match/info/{m_id}` — получить данные матча из базы
- `WS /match/ws/{uid}` — WebSocket для игрового процесса и поиска соперника

## WebSocket события

Клиент и сервер обмениваются следующими событиями:

- `start` — начать поиск соперника
- `cancel` — отменить поиск
- `matches_found` — найден соперник и создан матч
- `move` — сделать ход
- `board_update` — обновить доску после хода
- `game_over` — окончание матча
- `close_match` — завершить матч досрочно

## Структура базы данных

### Таблица `player`

- `uid` — уникальный идентификатор
- `nickname` — имя игрока
- `elo` — рейтинг ELO
- `wins` — количество побед
- `loses` — количество поражений
- `matches` — общее число матчей
- `datetime` — время регистрации
- `status` — текущий статус игрока (`online`, `offline`)

### Таблица `match`

- `mid` — идентификатор матча
- `player_1` — UID первого игрока
- `player_2` — UID второго игрока
- `winner` — UID победителя или пусто при ничье
- `elo_1`, `elo_2` — изменение ELO по результату
- `start_datetime` — время начала матча
- `finish_datetime` — время окончания матча

## Консольные команды сервера

В консоли сервера доступны:

- `/stop` — корректно остановить сервер и перевести всех игроков в `offline`
- `/rating [-e|-w|-l|-g] [-lim X] [uid]` — вывести рейтинг
  - `-e` — ELO
  - `-w` — победы
  - `-l` — поражения
  - `-g` — количество матчей
  - `-lim X` - можно задать количество выведеной информации(необязательно)
  - `uid` - при его вводе выводится информация определенного игрока(можно комбинировать с `-e`,`-w`,`-l`,`-g`)

## Рекомендации

- Убедитесь, что `xo-client.py` подключается к правильному `HOST_URL`.
- Если сервер запускается в Docker, настройте `HOST_URL` на `127.0.0.1:8000`.
- Для разработки можно добавить документацию OpenAPI от FastAPI, если потребуется расширение API.
